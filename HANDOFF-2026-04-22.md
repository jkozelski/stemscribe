## Top-of-stack problem you're inheriting

**The chord chart renders from vocal-aligned sections, not from the bar grid.** Alright by Jamiroquai illustrates it:

- True verse vamp: `Cm → Gm → Dm → Am` (4-bar loop × 13 in the song)
- Bass-anchored bar grid (correct): bar 1 = Cm, bar 2 = Gm, bar 3 = Dm, bar 4 = Am, …
- What renders today: verse starts with **Am** because the vocalist sings a pickup on the last Am before the Cm downbeat, and the section builder uses vocal-onset as the section boundary.

**The fix is a render-layer rewrite**, not a detector fix. The detector + bass extractor + quality smoother already produce correct per-bar chords. We just need to lay them out on screen starting at the bar grid's downbeat, with lyrics flowing under the bars. I got as far as attaching `bar_grid` to the chord-chart JSON. Rendering it is the next team's job.

**Evidence this is a render-only issue** — the backend output file `/opt/stemscribe/outputs/<jobId>/chord_chart.json` has a `bar_grid` array:

```json
"bar_grid": [
  { "bar": 1, "chord": "Cm", "start_time": 39.87, "end_time": 42.12,
    "bass_root": "C", "detector_quality": "m", "source": "bass+detector" },
  { "bar": 2, "chord": "Gm", "start_time": 42.12, "end_time": 44.40, ... },
  { "bar": 3, "chord": "Dm", ... },
  { "bar": 4, "chord": "Am", ... },
  { "bar": 5, "chord": "Cm", ... },
  ...
]
```

That grid is correct. The render doesn't use it — it uses the `sections[].lines[]` layout which is keyed on vocal timestamps. Rewrite the render to walk `bar_grid` and place lyrics underneath their bars.
