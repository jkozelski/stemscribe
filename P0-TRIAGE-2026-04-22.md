## One-pass triage procedure

Drive this with the new agent, not solo:

1. **Open `git status` in a terminal split.** Keep it live.
2. **Work through the SHIP chunks above in order.** For each chunk:
   - `git diff <files>` to sanity-check
   - `git add <files>`
   - `git commit -m "<message above>"`
3. **Update `.gitignore` with the DO NOT COMMIT list.** Commit the `.gitignore` change.
4. **For VPS_VARIANTS:** diff each pair, decide, delete the loser, commit.
5. **For MYSTERY directories:** tell the agent what each one is, then either commit, delete, or gitignore.
6. **For PRE-SESSION modifications:** go file by file. Agent shows you the diff; you say "ship" / "trash" / "branch" / "skip." Commit the keepers in one or two logical chunks ("batch: pre-launch hardening" is fine).
7. **`git push origin main` (or your branch).** Baseline established.

At the end of this, `git status` should show either clean or only one of these:
- `?? .cloudflare_token` (gitignored)
- Binary logs being actively written

Everything else should be accounted for.

Then — and only then — spawn the P1 agent team.
